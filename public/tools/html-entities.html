What are HTML Entities?

HTML entities are special text sequences that represent characters that would otherwise be interpreted as HTML markup or are not easily typeable. They begin with & and end with ;. For example, &lt; represents <, &amp; represents &, and &copy; represents ©.

Using entities prevents browsers from misreading special characters as HTML tags, and allows authors to include characters not available on a standard keyboard.

How to Use the HTML Entity Encoder

  1. Paste text containing special characters into the Input pane.
  2. Choose Encode to convert characters to their entity equivalents, or Decode to convert entities back to characters.
  3. Copy the result from the Output pane.
  4. Use the Reference Table below to look up specific entity codes.

Example

Input: <script>alert('XSS')</script>

Encoded output: &lt;script&gt;alert('XSS')&lt;/script&gt;

When rendered in HTML, the browser displays the literal text rather than executing the script tag — essential for safely displaying user-submitted code snippets.

Common Use Cases
  • Escape user-generated content before inserting it into HTML to prevent XSS attacks
  • Display code samples in HTML without the browser interpreting the tags
  • Include special characters (©, ™, →, €) in HTML without relying on keyboard input
  • Decode HTML entities from an API or scraping result back to readable text

Frequently Asked Questions

What is the difference between HTML entities and URL encoding?

HTML entities encode special characters for safe rendering inside HTML documents (&lt; for <). URL encoding converts characters for safe transmission in URLs (%3C for <). Use HTML entities for HTML content, URL encoding for query parameters and URL components.

Do I need to encode all non-ASCII characters?

Not necessarily. Modern HTML documents declare UTF-8 encoding (<meta charset="UTF-8">), which allows most Unicode characters to appear as-is. Entities are most important for the five characters with special HTML meaning: <, >, &, ", and '.

What is the difference between named and numeric entities?

Named entities use a defined name: &amp;, &copy;. Numeric entities use a decimal (&#38;) or hex (&#x26;) code point. All three forms produce the same character; named entities are more readable while numeric entities work for any Unicode character.
