What is URL Encoding?

URL encoding (also called percent-encoding) converts characters that are not allowed in a URL into a safe format. Each unsafe character is replaced by a % followed by its two-digit hexadecimal ASCII code — for example, a space becomes %20 and an ampersand becomes %26.

The standard is defined in RFC 3986. Only letters (A–Z, a–z), digits (0–9), and the characters - _ . ~ are considered "unreserved" and safe to use as-is in any part of a URL. Everything else must be percent-encoded when used in query strings, path segments, or fragment identifiers.

How to Use the URL Encoder / Decoder

  1. Paste or type your text in the input area.
  2. Choose Encode or Decode mode using the tab at the top.
  3. Select the encoding mode: Component encodes everything except unreserved characters (use for query parameter values); Full URL preserves structural characters like / ? # & = (use for encoding a full URL).
  4. The result updates instantly — click Copy to copy to clipboard.

encodeURIComponent vs encodeURI

  • encodeURIComponent (Component mode) — encodes everything except A-Z a-z 0-9 - _ . ! ~ * ' ( ). Use this for individual query parameter keys and values.
  • encodeURI (Full URL mode) — additionally preserves URL structural characters (; , / ? : @ & = + $ #). Use this when you want to encode a complete URL without breaking its structure.

Frequently Asked Questions

Why does a space sometimes appear as + and sometimes as %20?

+ represents a space only in the application/x-www-form-urlencoded format used by HTML forms. In standard RFC 3986 percent-encoding, a space must be %20. Using + outside of form data can cause bugs — this tool always outputs %20 for safety.

Should I encode the entire URL or just query parameters?

Encode individual query parameter values using Component mode, then assemble the URL. Encoding the full URL with Component mode will also encode the :// and / characters, breaking the URL structure. Use Full URL mode only if you need to safely embed a complete URL as a single value.

Are URL encoding and Base64 encoding the same?

No. URL encoding preserves the original bytes and makes them URL-safe using %XX notation. Base64 encoding converts binary data into a text representation using 64 printable ASCII characters, which increases size by ~33%. They serve different purposes — use URL encoding for URL parameters, Base64 for embedding binary data in text formats.
