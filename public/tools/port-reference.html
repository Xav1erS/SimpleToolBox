What is Port Number Reference?

A searchable reference for well-known TCP/UDP port numbers. Lists services, protocols, and descriptions for all system ports (0–1023) and common registered ports.

How to Use Port Number Reference

  1. Search by port number or service name (e.g., "ssh", "mysql", "3306").
  2. See port usage, protocol (TCP/UDP), and service description.
  3. Filter by TCP, UDP, or both.
  4. Copy the port number for firewall rules or config files.

Understand Port Number Reference ▾
Example

Port 22 → SSH (TCP): Secure Shell remote access

Port 80 → HTTP (TCP): Web traffic

Port 443 → HTTPS (TCP): Encrypted web traffic

Port 3306 → MySQL (TCP): Database connections

Port Number Ranges
  • 0–1023: Well-known system ports — require root/admin to bind on Unix
  • 1024–49151: Registered ports — assigned by IANA for specific services
  • 49152–65535: Dynamic/ephemeral ports — assigned by OS for outbound connections
Why Use This Tool
  • Quick reference when configuring firewalls or network rules
  • Check if a port is already assigned before using it for a custom service
  • Essential reference for DevOps and security work
Frequently Asked Questions
Does it include Docker and Kubernetes common ports?

Yes — popular container and orchestration tool ports (2376, 6443, 10250, etc.) are included.

Can two services use the same port number?

Only if they use different protocols (one TCP, one UDP). Two TCP services cannot share a port on the same interface.
