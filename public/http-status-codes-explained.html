HTTP Status Codes Explained

Use this page to quickly understand what a response code means, whether the issue is on the client or server side, and which codes matter most in debugging and API work.

Status code groups

1xx informational

The request is still in progress. These are uncommon in normal browser debugging.

2xx success

The request succeeded. 200, 201, and 204 show up most often.

3xx redirection

The client needs to follow another URL or use cached content.

4xx client error

The request was invalid, unauthorized, missing, or rate-limited.

5xx server error

The server failed while trying to handle a valid request.

Fast debug checklist

  • If the code is 4xx, check the request first: auth, payload shape, URL, headers.
  • If the code is 5xx, check the server and logs first.
  • If the code is 301 or 302, verify where the redirect lands and whether that behavior is intended.
  • If the code is 429, add retry logic or backoff instead of hammering the endpoint.
