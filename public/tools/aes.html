Note: Uses AES-256-GCM with PBKDF2 key derivation (100,000 iterations, SHA-256). A random salt (16 bytes) and IV (12 bytes) are generated for each encryption. The output is a single Base64 string containing salt + IV + ciphertext. Your passphrase is never uploaded — all processing happens in your browser. To decrypt, you must use the same passphrase that was used to encrypt.

What is AES Encrypt / Decrypt?

AES Encrypt / Decrypt uses browser-side AES to protect plaintext with a passphrase and decrypt it again when you need the original content. It is useful for testing encryption flows, understanding IV and authentication tag behavior, and handling short secrets locally without sending them to a server.

How to Use AES Encrypt / Decrypt

  1. Choose whether you want to encrypt plaintext or decrypt ciphertext.
  2. Enter the text and the passphrase you want to use.
  3. Run the conversion and inspect the generated ciphertext or recovered plaintext.
  4. Copy the result for testing, demos, or local workflows.

Frequently Asked Questions
What is the difference between AES-CBC and AES-GCM?

AES-CBC provides confidentiality only and requires a separate MAC for integrity verification. AES-GCM provides both confidentiality and built-in authentication, making it the preferred choice for new systems.

Why does the ciphertext change every time I encrypt the same message?

A random IV (initialization vector) is generated for each encryption operation. This ensures that identical plaintexts produce different ciphertexts, preventing pattern analysis. The IV is stored alongside the ciphertext and used automatically during decryption.

How strong is AES-256?

AES-256 is considered computationally unbreakable with current technology. A brute-force attack would require more energy than the observable universe could provide; it is the gold standard for symmetric encryption.

Can I use this tool to encrypt files?

This tool is designed for text encryption. For file encryption, consider dedicated tools. However, you can encrypt Base64-encoded file content if needed.
