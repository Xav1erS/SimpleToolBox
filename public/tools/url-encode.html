What is URL Encoding?

URL encoding converts reserved or unsafe characters into percent-encoded sequences so they can be transmitted safely inside URLs. It is commonly used for query strings, path segments, redirects, and parameters that include spaces or special characters.

Why Use This URL Encoder / Decoder?

When building or debugging URLs, manually counting percent-encoded characters is error-prone. This tool encodes or decodes instantly in your browser — no form submission, no server round-trip. It also distinguishes between encodeURI (whole URL) and encodeURIComponent (single parameter) semantics, which most simple encoders do not.

  • Instant output: Results appear as you type, no button press needed.

  • No server round-trip: Encoding and decoding happen locally in the browser.
  • Developer-friendly: Useful for debugging query strings, redirect URIs, and API parameters.

Frequently Asked Questions
What is the difference between encodeURI and encodeURIComponent?

encodeURI is for whole URLs and leaves separators like :, /, ?, and # intact. encodeURIComponent is for individual parts such as query values and encodes far more characters.

Why is a space encoded as %20 or +?

%20 is standard percent-encoding. + is mainly used by application/x-www-form-urlencoded form submissions. This tool outputs %20 by default.

Can I decode a full URL at once?

Yes. Paste the full URL and decode it to inspect all percent-encoded parts at once.

What is application/x-www-form-urlencoded?

It is the encoding format used by HTML form submissions, where spaces may be represented as +. For general URL components, %20 is the safer standard form.
