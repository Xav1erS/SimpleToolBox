Note: Uses AES-256-GCM with PBKDF2 key derivation (100,000 iterations, SHA-256). A random salt (16 bytes) and IV (12 bytes) are generated for each encryption. The output is a single Base64 string containing salt + IV + ciphertext. Your passphrase is never uploaded — all processing happens in your browser. To decrypt, you must use the same passphrase that was used to encrypt.

What is AES Encryption / Decryption?

Encrypts and decrypts text using AES-256-GCM with PBKDF2 key derivation (100,000 iterations, SHA-256). All processing happens client-side — your data and passphrase are never sent to any server.

How to Use AES Encryption / Decryption

  1. Enter your plaintext message.
  2. Enter a passphrase (the stronger the better).
  3. Click Encrypt to get a Base64 ciphertext string.
  4. To decrypt: paste the ciphertext, enter the same passphrase, click Decrypt.

Example

Plaintext: Secret message

Passphrase: myStrongPassphrase123!

Ciphertext (Base64): abc123...= (contains salt + IV + ciphertext)

Decrypt with same passphrase → original message

Security Details
  • AES-256-GCM: authenticated encryption (confidentiality + integrity)
  • PBKDF2 with 100,000 iterations: key derivation from passphrase
  • Random 16-byte salt + 12-byte IV generated per encryption
  • Output is a single Base64 string containing salt + IV + ciphertext
Why Use This Tool
  • Encrypt sensitive notes or config values without installing software
  • All processing is local — nothing leaves your browser
  • Test AES key derivation and GCM authenticated encryption behavior
Frequently Asked Questions
Is this secure enough for sensitive data?

AES-256-GCM is industry-standard. Security depends entirely on passphrase strength — use a long random passphrase.

Can I decrypt on another system?

Yes — any system using the same PBKDF2+AES-256-GCM algorithm with the same passphrase can decrypt the output.
