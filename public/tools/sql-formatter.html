What is SQL Formatter?

Formats SQL queries with consistent indentation, keyword capitalization, and clause alignment. Supports MySQL, PostgreSQL, SQLite, and SQL Server dialects.

How to Use SQL Formatter

  1. Paste your SQL query.
  2. Select your database dialect (MySQL, PostgreSQL, etc.).
  3. Click Format.
  4. Copy the formatted query.

Example

Input: select id,name from users where id=1 and active=true

Formatted:

SELECT
  id,
  name
FROM users
WHERE id = 1
  AND active = TRUE
Formatting Conventions Applied
  • Keywords uppercased (SELECT, FROM, WHERE, JOIN, GROUP BY)
  • One clause per line with consistent indentation
  • Column lists aligned
  • Subqueries indented one level
  • CTE (WITH clause) formatted correctly
Why Use This Tool
  • Makes complex multi-join queries readable for code review
  • Standardizes SQL style across team files
  • Works on single queries or entire scripts
Frequently Asked Questions
Does it work with CTEs and window functions?

Yes — modern SQL syntax including CTEs (WITH), window functions (OVER), and JSON operators is supported.

Does it check for SQL errors?

Formatting normalizes whitespace and casing but does not validate query correctness against a database.
