What are HTML Entities?

HTML entities are encoded forms of characters that would otherwise be interpreted specially in HTML, such as <, >, &, and quotes. They are useful when you need to display markup literally or safely insert text into HTML contexts.

How to Use HTML Entity Encoder / Decoder

  2. Choose encode when you need HTML-safe output, or decode to get readable text back.
  3. Review named or numeric entity output depending on your use case.

Frequently Asked Questions
What is the difference between HTML entities and URL encoding?

HTML entities encode special characters for safe rendering inside HTML documents ( &lt; for < ). URL encoding converts characters for safe transmission in URLs ( %3C for < ). Use HTML entities for HTML content, URL encoding for query parameters and URL components.

Do I need to encode all non-ASCII characters?

Not necessarily. Modern HTML documents declare UTF-8 encoding ( <meta charset="UTF-8"> ), which allows most Unicode characters to appear as-is. Entities are most important for the five characters with special HTML meaning: < , > , & , " , and ' .

What is the difference between named and numeric entities?

Named entities use a defined name: &amp; , &copy; . Numeric entities use a decimal ( &#38; ) or hex ( &#x26; ) code point. All three forms produce the same character; named entities are more readable while numeric entities work for any Unicode character.
