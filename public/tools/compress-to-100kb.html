What is a Target-Size Image Compressor?

A target-size image compressor automatically adjusts JPEG/WebP quality to produce an output file at or below a specific file size. Unlike a simple quality slider where you manually guess the right setting, this tool uses a binary search algorithm — it tries different quality levels and converges on the highest quality that still fits within your target. This is ideal for forms, applications, and upload systems that impose strict file size limits.

How to Compress an Image to 100KB

  1. Drop your image onto the preview area, or click to browse and select a file.
  2. Select a target size preset (50KB, 100KB, 200KB, 500KB) or enter a custom value.
  3. Choose the output format: JPEG (universal) or WebP (modern, smaller).
  4. Click Compress — the tool automatically finds the optimal quality level.
  5. Review the result (output size, quality used, status) and click Download.

Understand Target-Size Compression (algorithm, use cases & FAQs)
Example

Compress a 4MB vacation photo to fit within a government form's 100KB limit:

  • Input: JPEG, 4,096 KB, 4032×3024 px
  • Target: 100 KB, JPEG format
  • Tool runs binary search: tries quality 50%, 25%, 37%, 43%, 40%...
  • Output: JPEG at ~40% quality, 97 KB — within the 100KB limit

The algorithm automatically finds the highest quality that still meets your constraint.

Common Use Cases
  • Compress profile photos to fit government portals with 100KB or 200KB limits
  • Reduce image size for job application systems that reject large uploads
  • Prepare photos for email attachments with strict size limits
  • Compress images for web hosting plans with bandwidth constraints
  • Reduce image sizes for bulk upload to cloud storage or CMS systems
Why Use This Target-Size Compressor
  • No upload — compression happens entirely in your browser; images are never sent to a server.
  • Automatic quality search — binary search algorithm finds the optimal quality; no manual guessing.
  • Custom target — enter any KB value, not just the presets.
  • JPEG and WebP — WebP achieves smaller sizes at equivalent quality; choose based on your needs.
Frequently Asked Questions

What if the image can't be compressed to my target size?

If the target is extremely small (e.g., 10KB for a high-resolution photo), the tool will compress to the lowest quality possible and show "Best effort — target not fully met." In that case, try resizing the image dimensions first using the Image Resizer, then compressing.

How does the binary search algorithm work?

The algorithm starts at quality 50%, checks if the output is above or below the target, then halves the search range: if too large, try lower quality; if small enough, try higher quality. After ~15 iterations it converges to the highest quality that produces a file within your target size.

Why does the output size sometimes exceed the target by a few KB?

JPEG compression is not perfectly predictable — the actual output size depends on image complexity. The algorithm tries to get as close as possible while staying under the target, but final blob size may vary by 1–3 KB. If strict compliance is required, set your target slightly lower (e.g., 95KB instead of 100KB).

Should I use JPEG or WebP to achieve the smallest file?

WebP generally produces smaller files at the same visual quality. Use WebP if the platform you're uploading to supports it. Use JPEG for maximum compatibility with older systems and image editors.
