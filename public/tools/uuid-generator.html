What is a UUID?

A UUID (Universally Unique Identifier) is a 128-bit label used to uniquely identify information in computer systems. UUIDs are standardized by RFC 4122 and formatted as 32 hexadecimal digits displayed in five groups: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.

The most common version is UUID v4, which is generated using random or pseudo-random numbers. The probability of generating two identical v4 UUIDs is astronomically small — roughly 1 in 5.3 × 10³⁶ — making them safe to use as unique identifiers without a central registry.

Common Use Cases
  • Database primary keys — use UUID instead of auto-increment integers for distributed systems.
  • Session and token IDs — generate unpredictable session identifiers for security.
  • File naming — avoid collisions when saving user-uploaded files.
  • Microservices — correlate requests across distributed services using a shared trace ID.

Frequently Asked Questions

What is the difference between UUID v1 and v4?

v1 is time-based: it encodes the current timestamp and the machine's MAC address, making it sortable by creation time but potentially revealing system information. v4 is purely random and the recommended default for most applications.

Can two UUIDs ever be the same?

Theoretically yes, but the probability is negligible. For v4, with 2¹²² possible values, you would need to generate about 2.7 × 10¹⁸ UUIDs before having a 50% chance of a single collision.

Is UUID the same as GUID?

Yes. A GUID (Globally Unique Identifier) is Microsoft's term for the same RFC 4122 standard. The format and generation rules are identical.

Should I use UUID or auto-increment ID?

Use UUID when you need IDs generated on the client side, across multiple servers, or when you don't want sequential IDs to be guessable. Use auto-increment when you need simplicity and your database is a single instance.
