TOML (Tom's Obvious Minimal Language) is a config file format used by Cargo, uv, pyproject.toml, and many CLI tools. Use this converter to inspect TOML configs as JSON or generate TOML from a JSON object. Note: TOML dates are preserved as ISO strings in JSON output.

What is TOML to JSON Converter?

TOML to JSON Converter is a free browser-based tool that converts TOML configuration files to JSON and back, making it easy to work with Rust, Go, and other TOML-based config systems. All conversion runs locally in your browser — no data is ever uploaded.

How to Use

  1. Select the conversion direction: TOML → JSON or JSON → TOML.
  2. Paste your TOML or JSON into the left panel.
  3. The converted output appears instantly in the right panel — click Copy to use it.

Example

Converting a TOML database config section to JSON:

Input (TOML) [database]
host = "localhost"
port = 5432
Output (JSON) {"database":{"host":"localhost","port":5432}}

TOML's section headers map to nested JSON objects, and values preserve their native types.

TOML vs JSON vs YAML
  • TOML — designed for config files; minimal syntax, strongly typed, easy to read by humans. Common in Rust (Cargo.toml) and Go projects.
  • YAML — human-readable, supports comments and complex structures, widely used in DevOps (Docker, Kubernetes).
  • JSON — the universal data interchange format; no comments, strict syntax, supported everywhere.
  • When to use TOML — best for application configuration files where clarity and type safety matter.
  • When to use JSON — best for APIs, data storage, and any context where broad language support is needed.

Frequently Asked Questions

Are TOML comments preserved when converting to JSON?

No. JSON does not support comments, so TOML comments are dropped during conversion to JSON.

Does the tool support TOML arrays of tables?

Yes. TOML arrays of tables (double-bracket sections like [[products]]) are converted to JSON arrays of objects.

What TOML version is supported?

The tool supports TOML v1.0, which covers all standard features including dotted keys, inline tables, and multi-line strings.

Is my configuration data safe?

Yes. All processing happens entirely in your browser using JavaScript. No data is ever sent to a server.
