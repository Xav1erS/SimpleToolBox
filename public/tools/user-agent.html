User Agent Parser

Parse and decode User Agent strings to identify browser, OS, device, and engine details.

User Agent String
Note: UA string parsing is heuristic-based. Spoofed or custom User Agents may not be identified correctly. All processing happens entirely in your browser — no data is sent to any server.

What is User Agent Parser?

Parses User-Agent strings to identify browser name and version, operating system, device type (mobile/desktop/tablet), rendering engine, and bot detection flags. Useful for debugging analytics and server logs.

How to Use User Agent Parser

  1. Paste a User-Agent string, or click Use My UA to auto-detect your own.
  2. See browser, OS, device type, and engine broken down.
  3. Check the mobile and bot detection flags.
  4. Copy results for logging or debugging.

Understand User Agent Parser
Example

UA: Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15...

→ Browser: Safari 17, OS: iOS 17, Device: Mobile, Engine: WebKit

User Agent String Components

UA strings follow a legacy format for historical compatibility. Key tokens: browser identifier (Chrome/119.0, Firefox/120.0), OS (Windows NT 10.0, Intel Mac OS X), mobile signals (Mobile, Android, iPhone), engine (WebKit, Gecko, Blink), and bot markers (Googlebot, Bingbot).

Why Use This Tool
  • Debug analytics showing unexpected browsers or OS
  • Identify bots and crawlers in server access logs
  • Understand what device a test request came from
Frequently Asked Questions
Can User-Agent strings be trusted?

No — they can be spoofed by any client. Use UA parsing for analytics and soft detection, not for security decisions.

Does it detect bots?

Yes — common crawler bots (Googlebot, Bingbot, AhrefsBot, etc.) are flagged automatically.
